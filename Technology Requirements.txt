Functional Requirements
1. Authentication Module
Implement:

Student Registration

Student Login

Logout Functionality

User data must be stored securely in PostgreSQL.

2. Chapter Content Managemen
Select one academic chapter (for example, Class 10 Science – Life Processes, or any equivalent chapter).

Create:

Chapter record

Structured content storage

Topic-wise organization

Store and retrieve content from the database via backend APIs.

3. Learning Experienc
Create a chapter study page containing:

Chapter Overview

Topic-wise Learning Sections

Key Concepts

Revision Notes

The content should be rendered dynamically from backend APIs.

4. MCQ Assessment Module
Create a minimum of 20 MCQs for the selected chapter.

Each MCQ must contain:

Question

Four Options

Correct Answer

Explanation

Questions must be stored in the database and retrieved through APIs.

5. Online Examination Workflo
Implement a complete assessment flow:

Start Assessment
Create assessment session

Store session details in database

Assessment Interface
Display questions

Next / Previous navigation

Answer selection

Timer

Auto-save answers

Submission
Calculate score

Calculate percentage

Calculate accuracy

Store all results in the database.

6. Performance Dashboard
Create a student dashboard displaying:

Total Attempts

Scores

Accuracy

Correct Answers

Incorrect Answers

Skipped Questions

Chapter Completion Status

All information should be fetched from the database via APIs.
